# 개요
Plan Your Time Schedule.
입력한 데이터에 적합한 대학교 시간표를 자동으로 스케쥴링하는 프로그램.
대학생의 시간표가 아닌 대학교 특정 학과의 시간표를 스케쥴할 수 있다.

# 용어 설명
분반: 같은 교과목코드를 가진 강의로, 담당하는 전임교원만 다른 경우를 의미한다.
TP: Time Partition. 대학교 특정 강의의 시간 분할을 의미한다. 예를 들자면, 3시간 강의를 2시간과 1시간으로 분할하는 경우를 얘기한다.
TP Group: 시간표 상에 같은 시간에 배정어야만 하는 교과목들의 TP 그룹
교시: 특정 강의가 시작되는 시간. 0교시는 9시부터 10시까지를 의미한다.
강의 시간: 식별자 단위로 구분되는 각 강의의 교시부터의 실제 시간. 강의가 종료되는 시점은 강의 시간에 포함되지 않는다.
교과목: 교수가 강의 가능한 실제 교과목.
강의: 교수가 특정 교과목을 가르치는 것 또는 행위 자체.

# 알고리즘 설명
사용자가 입력한 교과목, 강의실, 전임교원 데이터를 토대로 제약조건에 부합하는 시간표를 생성해낸다.
처음 입력된 데이터에 대하여 모든 강의가 각각 배정 가능한 시간(날짜, 교시)과 장소(건물명, 강의실) 조합을 생성하여 저장한다.
여기에서 수십개의 강의마다 각각 약 또 수십개 내외의 조합이 발생한다.
그로 인해 해당 조합을 모두 컴토하는 경우 대략 50^50의 소요가 발생한다.
기본적으로 적용되는 재귀적 Backtracking 알고리즘은 부합하는 모든 시간표 조합을 검토하지만 해당 부하와 소요를 감당할 수 없다.
그러므로 지정된 시간 내에 생성되지 않으면 해당 경우의 경로를 폐기하고 임의적으로 재실행한다.
한 번 시간표가 완성되면 완성된 시간표에 대하여 데이터 프레임을 생성하고, 생성된 데이터 프레임을 엑셀 파일로 추출한다.

-대학교의 한 강의를 구별하는 식별자는 (교과목코드, 분반, TP)이다.
-클래스에 포함된 모든 변수는 한 개이상의 제약조건에 필수적이다.
-본 알고리즘은 시간표가 완성될 수 없는 것에 대해 보장하지 않는다.

# 제약조건 종류
1.한 교과목이 시간과 공간이 동시에 겹쳐서 배치될 수 없다.
2.한 교수는 하나의 강의를 진행하는 동안 다른 강의를 배정할 수 없다.
3.한 TP 그룹에 포함된 교과목 간에는 같은 시간에 배치되어야 한다.
4.학과 내 전공 필수 교과목 간에는 시간이 겹쳐서 배치될 수 없다.
5.같은 학년을 가진 교과목 간에는 시간이 겹쳐서 배치될 수 없다.
6.같은 교과목코드와 분반을 가지면서 다른 TP인 교과목끼리는 서로 다른 날짜에 배치되어야 한다. 즉, 시간이 분할된 교과목을 다른 날짜에 배치해야 한다.
7.해당 학과의 교수로 지정된 전임교원은 연구일을 제외한 나머지 4일에 최소 한 개이상의 교과목이 배치되어야 한다.
8.각 학년별로 점심시간이 존재해야 한다. 여기에서 점심시간의 정의는 11시에서 15시 사이의 공백을 의미한다. 

# 기타 제약사항
제약조건 중 8 번의 경우 시간표가 채워진 후에 검토하므로 부하와 소요가 크다.
모든 제약조건을 적용하여 시간표가 오랜 시간 나오지 않으면 8번 제약조건을 제외하는 것을 권장한다.
